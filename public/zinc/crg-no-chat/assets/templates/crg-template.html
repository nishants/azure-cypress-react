About It

  • A simple version of Close Reading Game.
  • It can create a new game, edit a game and play a game.
  • Code for this implementation is production quality and reusable in the Zinc Learning Lab platform.

About the Design

In this context, design includes

  • Data structures and models to support any complexity of scenes and interactions
  • Modeling data on web-server and database
  • Capturing user actions and saving it on a web-server
  • Frontend code for CRG implementation

Design here does not include the visual design. The styling is only placeholder and has to be redone based on mockups.

Model and Data Structure

Player
  • has state
  • runs script

Script
  • uses data to generate a sequence of scenes

Scene
  • collection of states

State
  • defines what get rendered in a scene
  • a scene can have one or more state
  • if a scene needs different elements to be rendered, its better to put them in different states to keep them small and testable.

UI Template
  • uses player state to render user interface
  • highlight text
  • focus on text
  • transcript dialog
  • buttons and actions
  • input box
  • action on text select
  • multi choice questions

Scenes as State Machine
  • Each scene can have one or more states
  • UI is rendered based on current scene and its state
  • Player moves from one state to another based on user actions
  • A state can be easily added, changed or removed without affecting any other state
  • Breaking a scene into states is recommended to keep it modular and testable
  • For example multi choice question scene has ask-question and show-result states
